## Accessibility

Recommended attributes to be included:

### aria-disabled

Indicates that the element is perceivable but disabled, so it is not editable or
otherwise operable. It is necessary to add it when the component is disabled.
With the 'aria-disabled' attribute the assistive technologies are capable of informing the user
that the link is disabled, but the click is not blocked, so it's necessary to avoid adding the
'href' attribute, specifying that it's still a link by the 'role' attribute.

<Source
  code={`
<a class="link" role="link" tabindex="0" aria-disabled="true">
    Disabled Link
</a>
  `}
/>

### tabindex

Users who do not or cannot use pointing devices can tab through links and, as
such, links should be in a logical tabbing order. The tabindex attribute allows
you to define this order although if the HTML is linear, as it should be, a
logical tabbing order should automatically fall into place.

<Source
  code={`
<a class="link" href="https://www.devo.com/" rel="noreferrer" target="_blank" tabindex="1">
    Link
</a>
<a class="link" href="https://www.devo.com/" rel="noreferrer" target="_blank" tabindex="3">
    Link
</a>
<a class="link" href="https://www.devo.com/" rel="noreferrer" target="_blank" tabindex="2">
    Link
</a>
  `}
/>
